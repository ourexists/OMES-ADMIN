# OMES 工业化设备管理系统

## 一、项目概述

**OMES** 是一套面向工业制造场景的**边缘侧 MES（制造执行系统）** 与管理平台，用于场景设备连接、数据采集、产线/工单/物料/计划管理及多租户运维。支持
OPC UA、Modbus、MQTT、REST、S7、WINCC 等多种工业协议，实现“边缘采集 + 统一门户”的管理方式。

> 更多说明见：<https://blog.ourexists.site/2026/01/30/omes/>

---

## 二、项目架构

项目采用 Maven 多模块结构，各业务域独立成模块，通过统一门户或采集端对外提供服务。

### 2.1 顶层模块

| 模块           | 对应业务                           |
|--------------|--------------------------------|
| omes-device  | 场景、设备、设备组、采集与 SCADA 配置、运行/告警数据 |
| omes-line    | 产线、工位                          |
| omes-mat     | 物料主数据、BOM                      |
| omes-mo      | 工单及明细                          |
| omes-mps     | 主生产计划、计划与产线/工位关联、手册            |
| omes-sync    | 连接配置、同步任务、设备与采集通道绑定            |
| omes-task    | 定时任务、配方                        |
| omes-ucenter | 租户、账号、角色、权限、部门、平台              |
| omes-message | 消息、通知                          |
| omes-portal  | 管理端 Web 应用（可执行）                |

### 2.2 可运行应用

- **管理端（portal-ms）**：聚合上述各业务模块，提供完整 Web 管理界面，默认端口 10010。

### 2.3 架构关系

管理端作为统一入口，调用设备、产线、物料、工单、计划、连接、任务、用户中心、消息等业务能力；采集端可独立部署，将现场数据通过 MQTT
等上报或与主系统同步，形成「边缘采集 + 集中管理」的架构。

```
                    ┌─────────────────────────────────────────┐
                    │              管理端 (portal-ms)          │
                    │        设备 · 产线 · 工单 · 计划 · 用户    │
                    └───────────────────┬─────────────────────┘
                                        │
    ┌──────────┬──────────┬──────────┬──┴──┬──────────┬──────────┐
    ▼          ▼          ▼          ▼     ▼          ▼          ▼
  设备       产线       物料      工单   计划      连接/任务   用户/消息
 (device)   (line)    (mat)     (mo)  (mps)    (sync/task)  (ucenter/message)

```

## 三、业务功能

以下按系统菜单与页面划分，对应管理端（portal-omes/portal-ms）实际提供的业务功能。

### 3.1 原料中心

- **原料信息**：物料主数据列表与维护（新增/编辑），支撑生产计划与工单用料。
- **原料分类**：物料分类维护，用于原料信息归类与筛选。

### 3.2 生产中心

- **计划流程**：主生产计划（MPS）的编制、计划明细、执行顺序与生产队列管理；支持加入/移出队列、插队置顶等。
- **生产订单**：制造工单（MO）列表、新增与编辑，订单投产与预期执行时间管理。
- **配方清单**：BOM 树形维护、BOM 及 BOMC/BOMD 编辑，配方成分与数值管理。
- **产线工艺**：产线及工位维护、产线工作流配置、数据同步时间与 PLC 下载时间、吞吐量与生产间隔等工艺参数。
- **设备工艺**：设备组/工艺卡维护，工艺与产线、流程的关联。
- **人工检测**：质量检测记录（检测状态、检测结果、合格量、检测描述等），支持人工录入与查看。

### 3.3 设备产线

- **场景管理**：车间/场景的列表与编辑，场景层级与归属；支持场景采集配置、场景组态（SCADA）入口。
- **设备管理**：设备台账（设备名、设备码、类别、定位、容量等）、设备编辑、设备详情、设备属性配置。
- **运行记录 / 报警记录 / 在线记录**：设备运行记录、报警记录、在线记录表格查询与展示。
- **实时设备**：设备实时状态与采集数据展示。
- **设备 GIS**：设备在地图上的定位与展示（经纬度）。
- **设备运营**：设备运营相关入口，包含：
    - **概览**：首页 KPI（设备总数、在线/离线、报警、运行/停机等）及设备趋势图表。
    - **场景组态**：场景级 SCADA 组态配置与展示。
    - **设备配置**：设备级采集或参数配置。
    - **场景报表**：按场景的统计报表。
- **大屏展示 / 设备大屏**：大屏看板与设备大屏页面，用于现场展示与监控。

### 3.4 数据管理

- **数据同步**：同步事务列表、同步资源与资源点、同步状态、断点重入；同步表单编辑。
- **连接管理**：多协议连接卡片/列表（服务名、连接地址、端口、协议、认证方式等），连接新增与编辑，采集配置入口。
- **任务管理**：定时任务列表与编辑（任务名称、任务种类、cron 表达式、任务状态）。

### 3.5 系统中心

- **账户管理**：账号列表、账号新增/编辑、平台/归属维护、账号与角色/场景分配。
- **系统角色**：角色列表、角色编辑、角色权限分配。
- **系统权限**：权限（API/菜单）列表与编辑。
- **消息通知**：通知列表与编辑（标题等），用于站内消息与告警推送。

### 3.6 统计与报表

- **原料统计**：原料维度统计展示。
- **产量统计**：产量维度统计展示。
- **过程报表**：生产过程相关报表。
- **能耗报表**：能耗相关报表（若有配置）。
- **场景报表**：按场景的统计与报表（见设备产线 · 设备运营）。

## 四、应用场景

| 场景          | 说明                                                 |
|-------------|----------------------------------------------------|
| **离散/流程制造** | 在场景边缘部署 OMES，统一管理产线、工单、计划、物料与设备，形成边缘 MES 能力。       |
| **多协议采集**   | 对接 PLC、SCADA、传感器等，通过 OPC UA、Modbus、MQTT 等协议采集设备数据。 |
| **集团多厂区**   | 多租户隔离，支持多工厂、多场景在同一套系统中分别使用与运维。                     |
| **看板与大屏**   | 与 SCADA、看板系统集成，提供实时采集数据接口，用于场景大屏、监控看板。             |

## 五、产品形态

- **管理端**：提供 Web 管理界面，覆盖设备、产线、工单、物料、连接、采集、SCADA、消息、用户等完整管理功能，是“一站式”边缘管理入口。
- **采集/同步端**：可独立部署的数据采集或同步服务，通过 MQTT 等协议接收或上报数据，适合现场分布式采集场景。
